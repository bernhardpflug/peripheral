JDDAC Documentation Trailmap

The Entazza Crew

This note describes the documentation available in the JDDAC package. If you are just getting started with JDDAC, depending on what you would like to do with JDDAC, this trail map will provide some guidance on what documentations you may wish to read.

If you would like to learn about the JDDAC architecture

Read the following articles which describe the architecture of the JDDAC APIs.

q       JDDAC Overview – A technical introduction to the family of JDDAC APIs.

q       JMDI Overview – A description of the JMDI architecture, including the following topics:

q       JTI Overview – A technical description of the JTI architecture, including the following topics:

If you would like to run the sample measurement application

To get started, read the brief introduction to JDDAC project, and perhaps the release notes for the latest code release.

Download the pre-compiled binary distribution package from the download page.

Read the installation instructions for the system requirements and to learn how to install the distribution after you download it

To run the sample application, how to access and use the measurement portal where the measurements are being archived, read the sample application getting-started guide.

If you would like to build on the sample measurement application for your own applications

Download the source code distribution package from the download page.

Read the installation instructions for the system requirements and to learn how to install the distribution after you download it

Read the build instructions and try to compile the source code to make sure that your environment is set up correctly. You may also want to read this explanation of the directory layout to learn how the files are organize within the JDDAC directory tree.

Lastly, the following documents will be helpful to developers.

q       Configuring a JDDAC system – The creation of JDDAC components, their configuration, and the wiring of these components into a dataflow application is controlled via a XML configuration file. This document explains how to modify the configuration file to set up your own applications. With the information in this document, you will be able to add your own components, change the wiring tolopology, or change the settings of the existing components.

q       Adding a F-Block – JDDAC applications are built from Function Blocks, or F-Blocks for short. This guide shows you how to write your own F-Block and add it to the system.

q       Add a Transducer – JDDAC applications interface with the physical world via Transducer objects. This guide shows you how to write your own Transducer object for a custom measurement and add it to the system.

q       FAQ - This is a list of frequently asked questions and answers which will help you go to the right sections in the Javadocs to find the development answers you seek.

q       Javadocs – These are the class and package level documentation generated from the comments in the source code.
