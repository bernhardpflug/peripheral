JDDAC Directory and Namespace Structure

Jerry Liu

Overview

This note describes the JDDAC directory structure.

These are the top level directories:

q       apps – Contains applications built from the JDDAC APIs.

q       common – Contains source code usable on all platforms: J2EE, J2SE, and J2ME.

q       docs – Contains documentation and javadocs.

q       lib – Contains the compiled JDDAC libraries.

q       lib-ext – Contains the third party library packages used by JDDAC.

q       portal – Contains the source code for portal (server) nodes running on J2EE.

q       probejme – Contains the source code for the probe nodes running on J2ME.

q       probejse -  Contains the source code for the probe nodes running on J2SE.

q       tools – Contains miscellaneous tools used in building JDDAC.

Within the apps directory, there is a subdirectory for each standalone JDDAC application.

q       Within the source code directories (common, portal, probejme, probejse), there is a src directory which contains the source code, and a build.xml ant build file for building the source code in the src directory.  The output of the compilation is typically stored in a bin directory which is dynamically created.

Within the lib-ext directory, there is a subdirectory for each third-party package.

Namespace

All Java source files are in the namespace net.java.jddac.*.

There are some classes which are used by all JDDAC APIs. These include classes that represent the common data structures and classes that provide basic utilities such as logging. These classes are found in net.java.jddac.common.*

The classes which make up the Java Measurement Calculus Interface (JMCI) are in net.java.jddac.jmci.*.

The classes which make up the Java Measurement Dataflow Interface (JMDI) are in net.java.jddac.jmdi.*.

The classes which make up the Java Transducer Interface (JTI) are in net.java.jddac.jti.*.

All three of these packges have subpackages below them. Please see the javadocs descriptions for additional details.
